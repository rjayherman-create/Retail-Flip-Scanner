Build a mobile-first and desktop-friendly app that helps a user find Costco markdown items worth flipping on Facebook Marketplace.

1. Photo Scan
2. Public Web Check
3. Screenshot Upload

Do not treat scraping/web checking as the only main method.
The user should be able to start from any capture method.

IMPORTANT COMPLIANCE RULES
The app may include a Public Web Check feature, but it must follow these rules:
- Do not bypass Costco login.
- Do not automate Costco member login.
- Do not bypass CAPTCHA.
- Do not bypass bot protection.
- Do not use stolen, copied, or user-exported cookies.
- Do not use private Costco APIs.
- Do not hammer Costco with repeated requests.
- Do not pretend inventory is confirmed if the page cannot be read.
- If public data is blocked, show a clean fallback message and direct the user to Photo Scan, Screenshot Upload, or Manual Add.

Compliance footer text:
“This tool supports photo scans, screenshot uploads, manual entries, and public web checks. It does not bypass Costco login, CAPTCHA, membership checks, bot protection, or private systems. Public web results may fail if inventory is not visible without login. Prices and stock may change.”

FLIP DECISION ENGINE
Every item must be scored the same way regardless of capture method.

Score range:
0 to 100

Scoring factors:
- Markdown strength
- Brand demand
- Expected resale spread
- Giftability
- Local Facebook Marketplace demand
- Ease of storage
- Seasonality
- Category demand
- Niche risk
- Saturation risk
- Bulky item risk
- Condition risk
- Competition risk

Markdown rules:
- Price ending .97 = Costco markdown / clearance signal
- Price ending .00 = possible manager/local markdown
- Price ending .88 = possible markdown signal
- Price ending .99 = normal retail, weaker signal
- Star on tag or screenshot = possible discontinued/limited stock signal

Recommendation rules:

BUY:
- score 75+
- expected profit at least $7 per unit
- item is easy to store and sell locally
- item has strong brand or category demand

MAYBE:
- score 55 to 74
- profit exists but demand is uncertain
- item may be niche
- suggest buying only 1 test unit

SKIP:
- score below 55
- low spread
- niche item
- bulky item
- damaged item
- slow resale risk
- too much local saturation

Flip Decision output must include:
- Recommendation: BUY / MAYBE / SKIP
- Flip score
- Main reason
- Costco price
- Suggested Facebook list price
- Expected sale price
- Estimated profit per unit
- Suggested quantity
- Negotiation floor
- Risk notes
- Storage notes
- Best next action

Example:

BUY

Reason:
Strong LEGO demand, good Costco markdown price, easy gift item, and small enough to store.

Costco price: $14.97
List price: $25
Expected sale: $20–$22
Profit estimate: $5–$8 each
Suggested quantity: 2–4
Negotiation floor: $20
Risk: Marketplace may get flooded if many local Costco shoppers find the same clearance.

SCORING FUNCTION
Create a reusable scoring function:

scoreFlipItem(item)

Inputs:
- product_name
- item_number
- price
- markdown_code
- category
- visible_brand
- stock_status
- normal_retail_estimate
- local_demand_notes

Outputs:
- flip_score
- recommendation
- facebook_list_price
- expected_sale_price
- estimated_profit
- max_quantity
- risk_notes

Basic scoring logic:
- Start at 50
- Add 20 if markdown_code is .97
- Add 15 if category is LEGO or strong gift category
- Add 10 if item is small/easy to store
- Add 10 if expected profit is at least $7
- Add 5 if item is sealed/new
- Subtract 15 if niche theme
- Subtract 10 if bulky
- Subtract 10 if expected profit below $5
- Subtract 10 if stock appears common across many stores
- Clamp score between 0 and 100

Listing price logic:
- For low-cost LEGO at $14.97, suggest listing around $25
- Expected sale around $20–$22
- Bundle option: 2 for $45
- Negotiation floor should protect minimum profit
